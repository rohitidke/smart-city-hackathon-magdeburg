# KISS-MD Datasets — Handout

322 datasets from the **Kommunales Informations- und Statistiksystem
Magdeburg (KISS-MD)** — Magdeburg's municipal statistics portal —
re-published here as JSON for direct use in web dashboards.
Source: City of Magdeburg, Office for Statistics
(<https://statistik.magdeburg.de/KISS-MD/>).

> *Deutsche Fassung:* [README.md](./README.md).

## Contents

```
.
├── index.json                       Manifest of all datasets

├── xlsx/<category>/<dataset>.xlsx   Original KISS-MD export (same slug)
└── examples/                        Runnable example project
    ├── README.md                    →  Run instructions

    ├── python/plot.py               pandas + matplotlib (weather)
    └── python/mietspiegel.py        pandas + matplotlib (rent index)
```

Sibling dataset in the same bundle: [`../mietspiegel-2024/`](../mietspiegel-2024/)
(aggregated Magdeburg rents 2012–2026 by district).

- **322 datasets** across **13 categories** (population, economy, transport,
  energy, weather, education, housing & construction, labour market, health, …)
- **All datasets have been manually reviewed** (`labelSource: "human-reviewed"`);
  unreliable labels were set to `null` rather than guessed
- All paths are ASCII-slugified (e.g. `bevoelkerung/...`,
  `oeffentliche-ordnung/...`) — no umlauts or whitespace in URLs

> **Quick start:** runnable example project with JS + Python code under
> [`examples/`](./examples/README.en.md) — visualisations from the weather
> and rent-index datasets, side by side.

## Schema

### Manifest `index.json`

```jsonc
{
  "datasetCount": 322,
  "categoryCount": 13,
  "categories": [{
    "slug": "wetter", "name": "Wetter", "count": 2,
    "datasets": [{
      "id":          "wetter/witterungsverhaeltnisse-monatlich",
      "title":       "Witterungsverhältnisse - monatlich",
      "subcategory": "Witterung in Magdeburg",
      "tableId":     "QB_41",
      "rowCount":    1711,
      "columnCount": 22,
      "path":        "json/wetter/witterungsverhaeltnisse-monatlich.json",
      "xlsxPath":    "xlsx/wetter/witterungsverhaeltnisse-monatlich.xlsx"
    }]
  }]
}
```

### Single dataset

```jsonc
{
  "id":          "wetter/witterungsverhaeltnisse-monatlich",
  "title":       "Witterungsverhältnisse - monatlich",
  "category":    "Wetter",
  "subcategory": "Witterung in Magdeburg",
  "labelSource": "human-reviewed",
  "labelNotice": "Column labels and units were manually verified against the raw data.",
  "description": "Monthly weather data for Magdeburg…",
  "columns": [
    { "key": "var1", "label": "Jahr",                        "unit": null, "type": "integer", "role": "year",     "sample": [2025, 2024, 2023] },
    { "key": "var2", "label": "Monat",                       "unit": null, "type": "string",  "role": "month_de", "sample": ["Juni","Mai"] },
    { "key": "var3", "label": "Lufttemperatur Monatsmittel", "unit": "°C", "type": "number",  "role": null,       "sample": [19.2, 14.1] }
  ],
  "rowCount": 1711,
  "rows": [
    { "var1": 2025, "var2": "Juni", "var3": 19.2 }
  ]
}
```

Top-level dataset fields:
- **`labelSource`** — `"human-reviewed"`, `"llm-draft"`, or `"none"`
- **`labelNotice`** — short note on how column labels were produced

Per-column fields:
- **`key`** — access key inside `rows[i]` (`var1`, `var2`, …)
- **`label`** — German column name; `null` when unknown or flagged unreliable
- **`unit`** — unit string (`°C`, `mm`, `%`, `Anzahl`, `EUR`, `1.000 EUR`, …) or `null`.
  All labels are in German for consistency with the source.
- **`type`** — `"integer"`, `"number"`, `"string"`, `"mixed"`
- **`role`** — optional `"year"` or `"month_de"` time-axis hint
- **`sample`** — first 3 non-`null` values for quick orientation

Missing values are encoded as `null` throughout.

## A note on labels

The KISS-MD source system publishes its tables without machine-readable
column names — internally columns are just `var1`, `var2`, … Since there is
no official label list, German labels and units were first **generated by a
language model from the title, category, and value samples**, then **all 322
datasets were manually reviewed**. The review was deliberately conservative:
column labels whose meaning could not be unambiguously derived from the
values and column headers were set to `null`. This is more honest than
guessing — `null` simply means "this column contains data whose precise
meaning we cannot guarantee."

Each dataset's `description` field documents what is known and which
columns were rejected and why — e.g. *"var3..var7 are 5 age groups whose
sum equals var8 (verified); the actual age boundaries could not be derived
from the raw data."* The `tableId` (e.g. `K8_T26`) can be looked up in the
original KISS-MD portal for clarification.

Columns with `label: null` still contain meaningful raw data — fall back to
`key`, `sample`, and the dataset's `description` text.

## Example 1 — JavaScript (Vanilla + Chart.js)

Line chart of monthly mean air temperature over the last two years:

```html
<canvas id="chart" style="max-width:720px"></canvas>
<script src="https://cdn.jsdelivr.net/npm/chart.js@4"></script>
<script type="module">
  const ds = await fetch('./json/wetter/witterungsverhaeltnisse-monatlich.json')
    .then(r => r.json());

  // Find the right columns by label (not by var-key)
  const yearCol  = ds.columns.find(c => c.role === 'year');
  const monthCol = ds.columns.find(c => c.role === 'month_de');
  const tempCol  = ds.columns.find(c => c.label === 'Lufttemperatur Monatsmittel');

  // Last 24 months, chronologically ascending
  const recent = ds.rows.slice(0, 24).reverse();
  const labels = recent.map(r => `${r[yearCol.key]} ${r[monthCol.key]}`);
  const values = recent.map(r => r[tempCol.key]);

  new Chart(document.getElementById('chart'), {
    type: 'line',
    data: { labels, datasets: [{ label: tempCol.label, data: values, tension: 0.3 }] },
    options: {
      scales: { y: { title: { display: true, text: tempCol.unit } } },
      plugins: { title: { display: true, text: ds.title } },
    },
  });
</script>
```

## Example 2 — Python (pandas + matplotlib)

```python
import json
import pandas as pd
import matplotlib.pyplot as plt

with open('json/wetter/witterungsverhaeltnisse-monatlich.json', encoding='utf-8') as f:
    ds = json.load(f)

# Rename var-keys to actual labels so the DataFrame is readable
rename = {c['key']: c['label'] or c['key'] for c in ds['columns']}
df = pd.DataFrame(ds['rows']).rename(columns=rename)

# Last 24 months, chronologically
recent = df.head(24).iloc[::-1].reset_index(drop=True)
recent['Zeitachse'] = recent['Jahr'].astype(str) + ' ' + recent['Monat']

temp_col = 'Lufttemperatur Monatsmittel'
ax = recent.plot(x='Zeitachse', y=temp_col, marker='o',
                 figsize=(10, 4), title=ds['title'])
ax.set_ylabel(next(c['unit'] for c in ds['columns'] if c['label'] == temp_col))
plt.xticks(rotation=45)
plt.tight_layout()
plt.show()
```

List every dataset of one category as a DataFrame:

```python

```

## Tips

- **Detecting time series** — columns with `role: "year"` / `role: "month_de"`
  are always the time axis. For pure cross-section tables (e.g. "crimes 2021
  by district"), `var1` still has `role: "year"` but every row carries the
  same year — check with `df['Jahr'].nunique()`.
- **Units live on the column** — read `column.unit` directly for axis
  labels instead of parsing them out of the label string.
- **Wide vs. long** — many tables are cross-tabs (e.g. population by age and
  gender: var1=Jahr, var2=Alter, var3..varN=age cohorts). Inspect the first
  few rows before applying `groupby`/`pivot`.
- **Sparse / unlabelled columns** — columns with `"label": null` are ones
  where the reviewer was uncertain. They mostly carry low-signal data; when
  in doubt, skip them.
- **Robust dataset selection** — always link via the `id`/`path` from
  `index.json`, never via the dataset title: titles can change, slugs are
  stable.
- **Language note** — all column labels and dataset titles are kept in
  German, matching the source. The schema field names (`key`, `label`,
  `unit`, `description`, `labelSource`, `labelNotice`, `rowCount`, etc.)
  are English so the data is easy to consume from non-German code.
